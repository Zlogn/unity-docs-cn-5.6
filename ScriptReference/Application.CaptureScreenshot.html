Application.CaptureScreenshot

Switch to Manual
public static void CaptureScreenshot (string filename, int superSize= 0);

Parameters

filename 截屏文件要保存到的路径名称。
superSize 分辨率的增加倍数。

Description

在路径 filename 捕获截屏并将其作为 PNG 文件。

如果该图像文件已存在，则会被覆盖。图像写入的位置 可以包含一个目录/文件夹列表。例如， 在 macOS 上，PNG 文件可以写入 /tmp/ScreenGrab.png。如果没有目录/文件夹 列表，图像将写入 Project 文件夹。CaptureScreenshot() 也可以 从 Editor 使用，例如在自定义 EditorWindow 中。默认情况下， 截屏图像也将写入 Project 文件夹。此外，必须选中 Game View ，以便 Editor 屏幕捕获正常工作。

当 superSize 参数大于 1 时，将生成更大分辨率的 截屏。例如，传递 4 将使截屏比通常情况大 4x4。 这适用于生成用于打印的截屏。

using UnityEngine;
using System.Collections;

public class ExampleClass : MonoBehaviour {
    void OnMouseDown() {
        Application.CaptureScreenshot("Screenshot.png");
    }
}

在移动平台上，文件名附加到持久数据路径。 请参阅 Application.persistentDataPath 以了解更多信息。 在 Android 上，此函数立即返回。生成的截屏将稍后提供。

Note: Screenshots can be made from the Editor as described above. In this case only the Game view can be captured. If the Scene view is selected any screenshot will not be written. Switching to the Game view will cause the most recent captured image to be written. See EditorWindow for how an editor-based capture window can be created. If the Game view is current and the Editor button is clicked multiple times the first generated screenshot will be written. Application.CaptureScreenshot is a Game specific function.
